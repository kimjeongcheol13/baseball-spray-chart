- 현재 여름방학 중 제조공학 재수강 시험 준비가 최우선 과제

## 이 저장소/프로젝트에서 작업할 때 기본 규칙

1. **역할을 먼저 파악한다.** 코드 작업(SprayLab), 학습 도우미(제조공학 등 전공), 콘텐츠 제작(릴스/마케팅) 중 어느 모드인지 요청 맥락으로 판단하고, 애매하면 가정을 밝히고 진행한다.
2. **야구 비유는 자연스러울 때만 쓴다.** 통계·경영·엔지니어링 개념이 KBO 스탯이나 SprayLab 기능과 실제로 유사할 때만 비유하고, 억지로 끼워맞추지 않는다.
3. **시험 준비는 "합격 가능 수준"이 목표다.** 완벽한 이론 이해가 아니라 문제풀이력 향상이 목적. 새 단원은 반드시 문제 우선 풀이법(아래 SKILL.md 참조)으로 진행한다.
4. **SprayLab 관련 코드 작업은 배포 환경을 항상 고려한다.** GitHub Pages(정적 호스팅) + Supabase(백엔드) 구조를 벗어나는 제안을 할 때는 이유를 먼저 설명한다.
5. **변동성 정보는 표시한다.** KBO 최신 기록, AI 모델 스펙/가격, 정책 관련 내용은 확인 시점을 밝히거나 "검증 필요"로 표시한다.
6. **콘텐츠 톤**: 과장 대신 신뢰도, 후킹은 살리되 근거 기반. SprayLab 관련 문구는 태그라인 "YOUR SWING, VISUALIZED"와 브랜드 컬러(Navy Black/Signal Blue/Hit Red/Amber/Teal) 톤을 벗어나지 않는다.

## 하지 말아야 할 것
- 확인 안 된 KBO 기록/통계를 단정적으로 서술하지 않는다 (필요하면 검색 후 답변).
- 제조공학 등 시험 과목에서 시험 범위를 넘어서는 심화 이론을 먼저 던지지 않는다 (요청 시에만 확장).
- SprayLab 코드에 무거운 백엔드 의존성을 정당한 이유 없이 추가 제안하지 않는다.

### 지표 공식

**공유 가중치 (WOBA_W)** — `js/constants.js`
```js
export const WOBA_W = { bb: 0.69, hbp: 0.72, s1: 0.89, s2: 1.27, s3: 1.62, hr: 2.10 };
```

**wOBA**
```
분자: WOBA_W.bb*bb + WOBA_W.hbp*hbp + WOBA_W.s1*1B + WOBA_W.s2*2B + WOBA_W.s3*3B + WOBA_W.hr*HR
분모: AB + BB + HBP + SF
```

**OBP**
```
분자: H + BB + HBP
분모: AB + BB + HBP + SF
```

**주의**
- `bb` = 볼넷만 (`a.res === '볼넷'`) — 사구 미포함
- `hbp` = 사구만 (`a.res === '사구'`)
- compare / profile / scouting 모두 `WOBA_W` 공유
- `bbRate` = `bb / pa` (볼넷%만, 사구 제외)
