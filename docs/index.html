SprayLab

야구 타구 기록·분석·공유를 위한 서버 없는 PWA.
단일 index.html 파일로 완성된 오프라인 우선 웹 앱입니다.

🚀로컬 실행 방법

방법 A — 파일 직접 열기

Shell
git clone https://github.com/kimjeongcheol13/baseball-spray-chart.git
cd baseball-spray-chart

# macOS
open index.html

# Windows
start index.html
⚠️ 제한 사항
file:// 프로토콜에서는 Service Worker가 등록되지 않아 PWA·오프라인 캐시가 비활성화됩니다. 핵심 기능(타구 기록·통계·저장·공유)은 정상 동작합니다.

방법 B — 로컬 서버 (전체 기능)

Shell
python3 -m http.server 8080
# 브라우저에서 http://localhost:8080 접속
✅ 권장
Service Worker는 https:// 또는 localhost에서만 등록됩니다. 로컬 서버를 사용하면 PWA 설치, 오프라인 캐시, 공유 URL까지 모든 기능이 활성화됩니다.

🎨Canvas 레이어 구조

세 개의 <canvas>가 같은 위치에 겹쳐 렌더링됩니다.

Canvas ID역할pointer-events
#fldCanvas야구장 배경 (잔디·흙·파울라인) 정적 렌더. 클릭 이벤트 수신all
#hitCanvas타구 점 · 방향 선 동적 렌더 (타석 추가 시 갱신)none
#ovrCanvas핫존 히트맵 오버레이 (토글 가능, opacity transition)none
좌표 시스템
타구 위치는 0~1 범위의 정규화된 좌표(x, y)로 저장됩니다. 렌더링 시 x * FS로 픽셀 좌표로 변환합니다. FS는 캔버스 크기(기본 440px)입니다.

📦localStorage 스키마

키타입설명
sl_savesJSON 배열저장 경기 인덱스. [{key, label, ts}, ...] 형태로 목록 관리
sl_<timestamp>JSON 객체경기 전체 데이터. GAME + 중첩된 PLAYER / ATBAT / PITCHER 배열
sl_ob3"1"온보딩 완료 플래그. 없으면 신규 사용자로 간주해 튜토리얼 표시
sl_visited"1"첫 방문 플래그
⚠️ 용량 한계
브라우저 localStorage 한도는 약 5~10 MB입니다. 경기당 평균 20~100 KB이므로 수십~수백 경기까지는 문제없지만, 장기간 누적 사용 시 저장 실패가 발생할 수 있습니다.

🔗공유 메커니즘

서버 없이 경기 데이터를 URL 한 줄로 공유합니다. LZ-String 라이브러리를 사용해 JSON을 URI-safe Base64로 압축합니다.

흐름
// 공유 URL 생성
const json = JSON.stringify(payload);
const encoded = 'z' + LZString.compressToEncodedURIComponent(json);
const url = `${location.origin}${location.pathname}?game=${encoded}`;

// 수신자가 링크 열 때 복원
const json = LZString.decompressFromEncodedURIComponent(encoded.slice(1));
const payload = JSON.parse(json);
// → 배너로 "공유된 경기가 있습니다" 안내 후 불러오기

공유 URL에 포함되는 데이터:

홈·원정팀 이름 점수 전체 타석 기록 타구 좌표 구종·존·이닝
레거시 지원
?game=b... (btoa), #share=... (해시) 형식도 파싱 가능합니다.

🤔기술적 의사결정

왜 Vanilla JS인가?
빌드 단계가 없어 index.html 하나로 GitHub Pages에 즉시 배포 가능합니다. 프레임워크 런타임 없이 단일 파일만 로드하므로 불안정한 야구장 LTE 환경에서도 빠릅니다.
왜 localStorage인가?
계정 없이 즉시 사용 가능합니다. 모든 데이터가 본인 기기에만 저장되어 개인정보를 보호하고, 인터넷이 끊겨도 정상 동작합니다. 서버 운영비도 없습니다.
왜 단일 index.html인가?
파일 하나만 GitHub Pages에 올리면 배포 완료입니다. Service Worker도 단일 URL만 캐시하면 완전한 오프라인 앱이 됩니다.
왜 LZ-String인가?
서버 없이 공유하려면 데이터를 URL에 담아야 합니다. LZ-String은 일반 JSON 대비 약 60~70% 압축하여 URL 길이를 크게 줄입니다.

⚠️한계 & 로드맵

현재 구조의 한계

💾
localStorage 용량 (~5~10 MB)
장기간 대용량 사용 시 저장 실패. 자동 정리 로직이 없어 사용자가 수동으로 삭제해야 합니다.
📱
디바이스 단독
기기 간 동기화 불가. 팀원 A의 기록을 팀원 B가 실시간으로 볼 수 없습니다.
🔗
공유 URL 길이
300타석에 가까워지면 LZ-String 압축 후에도 URL이 수 KB에 달해 일부 메신저에서 문제가 생길 수 있습니다.
📄
단일 HTML 유지보수
489 KB HTML 파일 하나에 CSS·JS·HTML이 전부 존재합니다. 기능 추가 시 파일이 계속 커집니다.
